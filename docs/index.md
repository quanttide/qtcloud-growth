# 量潮增长云工作文档

这个平台的主要功能是帮助企业建立自己的增长策略。
以当前的示例为例。我们都知道该做AB测试，但是在没有基础设施的情况下做AB测试不太现实。因此有必要把企业面临的真实情境搞清楚以后再制定增长策略。
那么理论上就存在一个更基础的元增长方法。这样就需要平台提供这个能够找到增长策略的元策略。
这个元增长的定义是“关于增长的自洽性”，通过这个元增长策略找到量潮在不同阶段不同时期不同情境下的的增长策略。
所以平台要提供一套“如何发现自洽的增长策略”的机制。
首先要提问用户，了解当前真实的增长现状和困境是什么。
然后是引导用户说出自己的亮点。
接着是提出一些方向引导用户。以及给一个观察者负责判断、使用范畴论做提示词基础。
最后是确定增长策略。
上面这些是设计阶段。
然后是验证阶段。就是要不断地收集数据来看真实情况是如何的。
平台并不帮助用户解决增长问题，而是帮助用户看清自己的增长模式是什么。
帮助团队更好地统一认识协作。
需要的分析工具应当自然发展演化。
